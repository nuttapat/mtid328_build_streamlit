Part 1: Miniconda Setup & Command-Line 101

We will be building this application entirely on your local machine using Miniconda. To do this, we need to get comfortable with the Terminal (also known as the command line).

1. Open Your Terminal

Windows Users

Open your Start Menu and search for "Anaconda Prompt (Miniconda3)". Do not use the standard Command Prompt.

Mac Users

Open Spotlight (Cmd + Space) or Launchpad and search for "Terminal".

Command-Line 101 Cheat Sheet
Action Mac Command Windows Command Explanation
Where am I? pwd cd Shows your current folder path.
What's in here? ls dir Lists all files and folders in your current location.
Make a new folder mkdir <name> mkdir <name> Creates a new folder. e.g., mkdir mtid328_project
Go into folder cd <name> cd <name> Moves you inside a folder. e.g., cd mtid328_project

2. Set Up the Virtual Environment

A virtual environment keeps your project's Python libraries isolated so they don't mess up other projects. Run these two commands one by one in your terminal:

Command 1: Create the environment (named 'mtid328')

conda create -n mtid328 python=3.10 -y

Command 2: Activate the environment

conda activate mtid328

Check for success: Look at your terminal prompt line. You should now see (mtid328) at the very beginning of the line. If you see this, you are ready to code!

Part 2: Building the Machine Learning Model

First, use your terminal to create a project folder and navigate inside it. Then open your favorite text editor (such as Notepad++ on Windows or BBEdit on macOS) to create a file named train.py.

Goal: Install Necessary Libraries

Ensure your terminal is open and your (mtid328) environment is activated. We need to install the tools for data handling (`pandas`) and AI (`scikit-learn`).

Your Prompt for Gemini:

Give me the Python code to install the scikit-learn and pandas libraries using conda in a Terminal.

Run this in your terminal:

conda install scikit-learn pandas

What's Happening?

This command downloads and installs the two specified Python packages into your local virtual environment, making their functions available for us to use.

Goal: Load the Dataset

Open your empty train.py file in your text editor. We'll start by adding code to load the Pima Indians Diabetes dataset.

Your Prompt for Gemini:

I need to load the Pima Indians Diabetes dataset. Give me the Python code using pandas to load it from this URL and display the first 5 rows: https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.data.csv

The dataset does not have column names. Please add them in this order: 'preg', 'plas', 'pres', 'skin', 'test', 'mass', 'pedi', 'age', 'class'.

Expected Python Code:

Add this to train.py.

import pandas as pd url = "https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.data.csv" names = ['preg', 'plas', 'pres', 'skin', 'test', 'mass', 'pedi', 'age', 'class'] data = pd.read_csv(url, names=names) print(data.head())

Goal: Separate Data into Features (X) and Target (y)

Continue editing train.py. We need to split our data into `X` (clinical measurements) and `y` (final diagnosis).

Your Prompt for Gemini:

Using the pandas DataFrame from the previous step, show me how to separate it into features (X) and the target variable (y). The target variable is the 'class' column.

Expected Python Code:

Add this to train.py.

X = data.drop('class', axis=1) y = data['class']

Goal: Train the AI Model

Next, we "teach" the AI model. We split the data, using most of it for training and saving a small part for testing.

Your Prompt for Gemini:

Now, add code to create and train a Logistic Regression model using scikit-learn. Use the X and y variables. Split the data so 70% is for training and 30% is for testing.

Expected Python Code:

Add this to train.py.

from sklearn.model_selection import train_test_split from sklearn.linear_model import LogisticRegression X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42) model = LogisticRegression(max_iter=200) model.fit(X_train, y_train) print("Model trained successfully!")

Goal: Save & Run Your Code

Finally, we need code to save our trained model so our future web app can use it. After adding this, we will run the script.

Your Prompt for Gemini:

How do I save my trained scikit-learn model to a file named `diabetes_model.pkl` using the pickle library in Python?

Expected Python Code:

Add this to train.py.

import pickle filename = 'diabetes_model.pkl' pickle.dump(model, open(filename, 'wb')) print(f"Model saved to {filename}")

The Moment of Truth

Save your train.py file in your text editor. Go back to your terminal, ensure you are in the correct folder, and run the file:

python train.py

If successful, you will see output printing the data, and a new file named diabetes_model.pkl will magically appear in your folder!

Since Orange3 handles the data loading and training visually, you don't need to write Python code for this step! Instead, you will build a visual workflow. Open Orange3 and create a new project.

1. Load the Data (File Widget)

Drag the File widget onto the canvas. Double-click it, select 'URL', and paste this dataset link: https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.data.csv.

Crucial Step: Because the file doesn't have headers, Orange names them Feature 1, Feature 2, etc. Find the very last feature (the outcome) and change its Role from 'Feature' to 'Target'.

2. Split the Data (Data Sampler)

Drag the Data Sampler widget onto the canvas and connect it to the File widget. Double-click it and set it to sample 70% of the data for training.

3. Train the Model (Logistic Regression)

Drag the Logistic Regression widget onto the canvas and connect the Data Sampler to it. The model will automatically train on your data!

4. Export the Model (Save Model Widget)

Drag the Save Model widget onto the canvas and connect Logistic Regression to it. Double-click the Save Model widget, navigate to your empty project folder, and save the file exactly as:
diabetes_model.pkcls

Part 3: Building the Web App

Now that we have our saved AI model, we will build a user-friendly web interface for it using Streamlit.

Goal: Create the Streamlit App Script

Create a new, empty file in your text editor named app.py. Make sure it is saved in the exact same folder as your model file.

Your Prompt for Gemini:

Create a complete Python script for a Streamlit web app. It must do the following:

  1. Load a trained scikit-learn model from a file named `diabetes_model.pkl`.
  2. Display the title "Diabetes Prediction App".
  3. Create sidebar sliders for a user to input data for each of these features: 'preg', 'plas', 'pres', 'skin', 'test', 'mass', 'pedi', 'age'.
  4. Have a "Predict" button. When clicked, it uses the loaded model to predict the outcome based on the user's slider inputs.
  5. Display the final prediction in a clear message.

Create a complete Python script for a Streamlit web app that loads an Orange3 model. It must do the following:

  1. Load a trained Orange3 model from a file named `diabetes_model.pkcls` using pickle.
  2. Display the title "Diabetes Prediction App".
  3. Create sidebar sliders for a user to input data for 8 continuous features.
  4. Convert the slider inputs into an `Orange.data.Table` by splitting the data into features (X) and a dummy target (Y).
  5. Have a "Predict" button that uses the model to predict the outcome, and displays the result clearly.

Expected Python Code (`app.py`):

import streamlit as st import pandas as pd import pickle # Load the trained model with open('diabetes_model.pkl', 'rb') as model_file: model = pickle.load(model_file) st.title('🩺 Diabetes Prediction App') st.write("Enter patient data to predict the likelihood of diabetes.") # Create sidebar for user input st.sidebar.header('Patient Data') def user_input_features(): preg = st.sidebar.slider('Pregnancies', 0, 17, 3) plas = st.sidebar.slider('Plasma Glucose', 0, 200, 120) pres = st.sidebar.slider('Blood Pressure', 0, 122, 70) skin = st.sidebar.slider('Skin Thickness', 0, 99, 20) test = st.sidebar.slider('Insulin Test', 0, 846, 79) mass = st.sidebar.slider('Body Mass Index (BMI)', 0.0, 67.1, 32.0) pedi = st.sidebar.slider('Diabetes Pedigree Function', 0.078, 2.42, 0.3725) age = st.sidebar.slider('Age (years)', 21, 81, 29) data = { 'preg': preg, 'plas': plas, 'pres': pres, 'skin': skin, 'test': test, 'mass': mass, 'pedi': pedi, 'age': age } features = pd.DataFrame(data, index=[0]) return features input_df = user_input_features() st.subheader('Patient Input Data') st.write(input_df) if st.button('Predict'): prediction = model.predict(input_df) st.subheader('Prediction Result') if prediction[0] == 1: st.error('The patient is likely Diabetic.') else: st.success('The patient is likely Not Diabetic.')
import streamlit as st import pandas as pd import numpy as np import pickle import Orange # Load the trained Orange3 model with open('diabetes_model.pkcls', 'rb') as model_file: model = pickle.load(model_file) st.title('🩺 Diabetes Prediction App (Orange3)') st.write("Enter patient data to predict the likelihood of diabetes.") # Create sidebar for user input st.sidebar.header('Patient Data') def user_input_features(): preg = st.sidebar.slider('Pregnancies', 0, 17, 3) plas = st.sidebar.slider('Plasma Glucose', 0, 200, 120) pres = st.sidebar.slider('Blood Pressure', 0, 122, 70) skin = st.sidebar.slider('Skin Thickness', 0, 99, 20) test = st.sidebar.slider('Insulin Test', 0, 846, 79) mass = st.sidebar.slider('Body Mass Index (BMI)', 0.0, 67.1, 32.0) pedi = st.sidebar.slider('Diabetes Pedigree Function', 0.078, 2.42, 0.3725) age = st.sidebar.slider('Age (years)', 21, 81, 29) data = { 'preg': preg, 'plas': plas, 'pres': pres, 'skin': skin, 'test': test, 'mass': mass, 'pedi': pedi, 'age': age } features = pd.DataFrame(data, index=[0]) return features input_df = user_input_features() st.subheader('Patient Input Data') st.write(input_df) if st.button('Predict'): # Convert pandas DataFrame to Orange Table using the model's domain domain = model.domain # Orange expects features (X) and targets (Y) as separate arrays X = input_df.to_numpy() Y = np.zeros(1) # Dummy target required for prediction input_table = Orange.data.Table.from_numpy(domain, X, Y=Y) prediction = model(input_table) st.subheader('Prediction Result') # Orange returns the class index if prediction[0] == 1: st.error('The patient is likely Diabetic.') else: st.success('The patient is likely Not Diabetic.')

Part 4: Local Test

Before sharing with the world, we must test the app on our own computer.

Step 1: Terminal Check

Make sure you are still in your project folder in the terminal, and that your (mtid328) environment is active.

Step 2: Install Web Libraries

We need to install Streamlit to handle the UI, and the framework for our model.

conda install streamlit scikit-learn pandas
conda install streamlit orange3 pandas

Step 3: Run the App

Launch your app! This command tells Streamlit to read your app.py script and turn it into a website.

streamlit run app.py
🎉

Your web browser should automatically open a new tab with your local app. Test the sliders and click predict!

Part 5: Share with the World 🌍

Right now, the app only lives on your computer. To let others use it (like on their smartphones), we need to host it on the internet using GitHub and Streamlit Community Cloud.

Step 1: The Requirements File

The cloud server needs to know what Python libraries your app relies on. Open your text editor, create a new file, and save it as exactly requirements.txt in the same folder as your other files.

Paste this exact text into requirements.txt:

streamlit pandas scikit-learn
streamlit pandas orange3

You should now have 3 important files in your folder: app.py, diabetes_model.pkl, and requirements.txt.

You should now have 3 important files in your folder: app.py, diabetes_model.pkcls, and requirements.txt.

Step 2: Upload to GitHub

GitHub is a platform where developers store their code. We will use it to hold your files so Streamlit can find them.

  1. Go to GitHub.com and sign up for a free account.
  2. Once logged in, look for the + icon in the top right and click New repository.
  3. Name your repository something like diabetes-predictor. Scroll down and click Create repository.
  4. On the next screen, look for the small link that says "uploading an existing file".
  5. Drag and drop your 3 files into the box.
  6. Click the green Commit changes button at the bottom.

Step 3: Deploy on Streamlit Cloud

Now we link Streamlit to your GitHub so it can build the actual website.

  1. Go to share.streamlit.io and sign in. Choose "Continue with GitHub" to link your accounts.
  2. Click the New app button in the top right.
  3. You might need to grant Streamlit access to your GitHub repositories if prompted.
  4. Under "Repository", select your newly created diabetes-predictor repository from the dropdown.
  5. The "Main file path" should be set to app.py.
  6. Click Deploy!
🚀

You did it!

Wait a minute or two for the servers to bake your app. Once it loads, you can copy the URL from your browser's address bar and send it to anyone! You have successfully built and deployed a machine learning web app from scratch.
